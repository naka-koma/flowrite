計画に基づいてIssueの実装を行う。

## 手順

1. 直前の `/plan` で合意した計画、または `$ARGUMENTS` で指定された内容に従って実装する
2. 実装ステップごとに以下を行う
   - 該当ファイルの作成・編集
   - CLAUDE.mdのコーディング規約に従っているか確認
   - ステップ完了時に進捗を報告
3. 全ステップ完了後、ビルド確認を行う
4. 変更内容のサマリーを提示する

## ビルド確認（実装完了後に必ず実施）
- フロントエンドの変更がある場合: `cd frontend && npm run build` を実行してビルドが通ることを確認する
- TypeScriptの型エラーがないことを確認する
- ビルドエラーがあれば修正してから完了とする

## コーディング規約の確認ポイント
- フロントエンド: `any` を使っていないか、型定義が `types/` に切り出されているか
- GAS: `var` を使っていないか、スプレッドシート操作が一括処理になっているか
- 共通: エラーが握りつぶされていないか

## 注意
- 実装中にコミットは行わない（ユーザーの確認後にコミットする）
- 計画にない変更やリファクタリングは行わない
- 判断に迷う箇所があればユーザーに確認する
