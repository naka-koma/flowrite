# MoneyForward CSVフォーマット

## エクスポート方法

MoneyForwardの「収入・支出」→「収入・支出詳細」から期間を指定してCSVダウンロード。

## ファイル仕様

- エンコード: **Shift-JIS**（UTF-8ではない点に注意）
- 区切り文字: カンマ（`,`）
- 1行目: ヘッダー行
- 改行コード: CRLF

## カラム定義

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| 1 | 計算対象 | number | 1=集計対象、0=集計除外 |
| 2 | 日付 | string | YYYY/MM/DD形式 |
| 3 | 内容 | string | 店舗名・サービス名 |
| 4 | 金額(円) | number | 支出は負値、収入は正値 |
| 5 | 保有金融機関 | string | カード名・口座名 |
| 6 | 大項目 | string | カテゴリ（食費、交通費など） |
| 7 | 中項目 | string | サブカテゴリ |
| 8 | メモ | string | 任意メモ（空文字の場合あり） |
| 9 | 振替 | number | 1=振替取引、0=通常取引 |
| 10 | ID | string | ユニークID（重複排除に使用） |

## 取り込み時の注意事項

### Shift-JISのデコード
GAS側では以下のように処理する。

```javascript
const blob = Utilities.newBlob(data, 'application/octet-stream');
const text = blob.getDataAsString('Shift_JIS');
```

### 除外すべき行
以下の行はスプレッドシートに取り込むが、集計からは除外する。

- `計算対象 = 0` の行（ユーザーが集計除外に設定したもの）
- `振替 = 1` の行（口座間の資金移動）

### 重複排除
ID列（10列目）をキーとして重複を判定する。同じIDが `raw_data` シートに存在する場合はスキップする。複数月分のCSVを一括アップロードする際に重複が発生しうるため、必ず確認する。

### 金額の符号
- 支出: 負値（例: `-5742`）
- 収入: 正値（例: `620`）

スプレッドシートには符号をそのまま保持する。フロントエンドでの表示時は符号を考慮して絶対値に変換する。
